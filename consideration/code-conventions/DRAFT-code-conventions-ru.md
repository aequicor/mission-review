# Draft: code conventions

## Контекст задачи

В проекте есть `AGENTS.md`, который описывает роль агента, стек и процесс разработки, но отдельного блока code conventions пока нет. Пользователь попросил добавить такой блок в `AGENTS.md` или вынести его в отдельный skill.

## Цель изменения

Зафиксировать единые правила написания Kotlin/JVM/KMP-кода, coroutine-кода и KDoc-документации для будущей реализации `mission-review`.

## Предполагаемая архитектура

Правила оформляются как проектный блок в `AGENTS.md`, а не как отдельный Codex-skill.

Причины:

- правила относятся именно к этому репозиторию и должны читаться вместе с процессом разработки;
- `AGENTS.md` уже является точкой входа для обязательных требований агента;
- отдельный skill стоит создавать позже, если эти conventions нужно будет переиспользовать в нескольких проектах.

## Затрагиваемые модули

- `AGENTS.md` - добавляется блок `BLOCK: Code conventions`;
- `consideration/code-conventions/DRAFT-code-conventions-ru.md` - фиксируется решение и критерии готовности.

## Ограничения

- Не менять процесс commit и human review.
- Не добавлять tooling, formatter или linter без отдельного draft.
- Не копировать дословно большие фрагменты внешних руководств, а сформулировать проектные правила на их основе.
- Сохранять русский язык существующего `AGENTS.md`.

## Открытые вопросы

- Нужен ли в будущем отдельный reusable skill для тех же conventions в других репозиториях.
- Будет ли проект использовать `ktlint`, `detekt` или Gradle formatting task как автоматическую проверку этих правил.

## Критерии готовности

- В `AGENTS.md` есть отдельный блок code conventions.
- В блоке отражены правила организации Kotlin-кода, именования, форматирования, coroutine-практик и KDoc.
- Указаны исходные reference-документы.
- Изменение ограничено документацией и не меняет исходный код приложения.
